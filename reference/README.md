## Reference repositories

These repositories have full copies of working code; however, there is a lot of code here.

DO NOT READ THESE AS PART OF ANY GENERALIZED EXPLORATION.

DO READ them when

1. A Skill or Instruction specifically reference a particular file.
2. You need to understand a specific part of hyrax in detail (e.g. because it is in a stack trace or specific debug)

Even then, keep reads SPECIFIC and TARGETED at the problem or instruction you are currently working on.
